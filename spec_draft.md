SQLノートブックの仕様書
これはSQLノートブックの仕様書である。SQLノートブックはJupiterノートブックのようにSQLをマークダウン形式で記載し、実行ボタンでそれを実行し、結果を出力するものである。

SQLノートブックは以下の機能を持つ。
- SQLコードの記述
- SQLコードの実行
- SQLコードの結果の表示

## 実装方針

SQLノートブックは、シンプルなHTMLとJavaScriptを使用して実装し、Netlifyにデプロイする。
ページ遷移等のないシングルページアプリケーションとし、リッチな機能を提供することはせず、あくまでSQLコードの記述と実行に特化したシンプルなノートブックとする。

## SQLコードの実行環境

SQLノートブックは、TiDB cloud Zeroを利用してSQLコードを実行する。TiDB cloud Zeroは、TiDBのクラウドサービスであり、SQLコードを実行するための環境を提供する。

初回実行時に、TiDb Cloud Zeroの接続情報がなければ、クラスタを新規で作成して、接続情報をブラウザのローカルストレージに保存する。次回以降は、保存された接続情報を使用してSQLコードを実行する。TidB cloud zeroの有効期限が切れた場合も、同様に新しいクラスタを作成して接続情報を更新する。

## SQLノートブックの形式

SQLノートブックの実体は、Markdown形式のテキストファイルである。SQLコードは、Markdownのコードブロック内に記述される。

## SQLコードの記述

SQLコードはMarkdownのコードブロック内に記述される。コードブロックの言語指定は「sql」とする。例えば、以下のようにSQLコードを記述することができる。

```sql
SELECT * FROM users;
```

SQLコードを実行するためのボタンは、コードブロックの右上に配置される。ユーザーがこのボタンをクリックすると、対応するSQLコードがTiDB cloud Zeroで実行され、その結果がコードブロックの下に表示される。結果の表示形式は、テーブル形式とする。SQLコードの実行結果が複数行にわたる場合は、スクロール可能な領域内に表示されるようにする。

## 変数の利用

SQLノートブックでは、変数を利用することができる。変数は、Markdownのenvブロック内に定義される。例えば、以下のように変数を定義することができる。

```env
DATABASE_NAME=my_database
TABLE_NAME=users
``` 

定義された変数は、SQLコード内で使用することができる。例えば、以下のように変数を使用することができる。

```sql
SELECT * FROM ${DATABASE_NAME}.${TABLE_NAME};
```

変数はSQL実行前に展開され、展開されたSQLコードがTiDB cloud Zeroで実行される。変数の値は、ノートブック全体で共有されるため、複数のSQLコードブロックで同じ変数を使用することができる。

## ノートブックの保存場所

ノートブックはユーザーのブラウザのローカルストレージに保存される。ユーザーがノートブックを保存すると、Markdown形式のテキストファイルがローカルストレージに保存される。ユーザーは、保存されたノートブックを後で再度開くことができる。

ノートブックのアップロードははmarkdown形式ファイルのアップロードや、gist URLの指定にも対応するものとする。

## ノートブックの共有

ユーザーはノートブックを他のユーザーと共有することができる。共有はgistを利用して行われる。ユーザーがノートブックを共有すると、Markdown形式のテキストファイルがgistに保存され、そのURLが生成される。ユーザーは、このURLを他のユーザーと共有することで、ノートブックを共有することができる。

読み取り時にURLにパラメーターとしてgistのIDを指定することで、共有されたノートブックを読み込むことができる。例えば、以下のようなURLでノートブックを読み込むことができる。

```text
https://sql-notebook.netlify.app/?gist_id=1234567890abcdef
```

gist利用時の認証は、ユーザーが自身のGitHubアカウントで行う。ユーザーがノートブックを共有する際に、GitHubのOAuth認証を行い、認証されたユーザーのアカウントでgistを作成する。共有されたノートブックを読み込む際には、特に認証は必要ない。

## 編集機能

SQLノートブックは、簡易的なMarkdownエディタを提供する。ユーザーは、SQLコードブロックやenvブロックを追加、削除、編集することができる。編集された内容は、ノートブックのMarkdownファイルに保存される。ノートブックの保存は、ユーザーが明示的に保存ボタンをクリックすることで行われる。
